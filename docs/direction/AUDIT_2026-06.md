## False positives — investigated and dismissed

Recorded so they don't resurface in future audits:

1. **"Secrets committed to git."** False. `.env.*` is gitignored (`.gitignore:163`);
   full git history contains no `.env` file — only `.envrc`, a commented-out placeholder.
   Local `.env` files hold real secrets, which is where they belong.
2. **"Critical idempotency race" in `lib/idempotency.ts`.** Overstated. The
   claim-by-insert (`onConflictDoNothing`) plus atomic conditional reclaim is correct;
   concurrent retries get a clean 409, and non-expired `failed` keys are reclaimable
   (line 99), so retries after failure work.
3. **"Audit flush races Lambda freeze."** False by JS semantics — an `await` in a
   `finally` block resolves before the handler's promise does.

## Findings

### Medium

1. **S3 versioning disabled** on the images bucket (`infrastructure/lib/media-stack.ts:43`) —
   same-key overwrite/delete is unrecoverable user-data loss. *Mooted by migration (R2).*
2. **CloudTrail logs expire after 1 year** (`infrastructure/lib/cloudtrail-stack.ts:37`)
   while docs claim 7-year SOC 2 retention — one of them must change. *Mooted; archive
   CloudTrail before teardown.*
3. **`ALERT_EMAIL` has no fail-fast** — unset, CDK creates alarms with no destination. *Mooted.*
4. **buildspec SSM fetches fall back to empty strings** (`2>/dev/null || echo ""`) —
   missing params fail cryptically downstream. *Mooted (GitHub Actions).*
5. **Audit write failures are logged but not metric'd** (`src/node/lib/audit.ts`) —
   fire-and-forget is right, but a sustained DB outage silently holes the compliance
   trail. Emit a metric + alarm. **Carries over to Workers — fix regardless.**
6. **No reserved concurrency on critical handlers** (deliberate; alarmed at 70% of
   account limit). *Mooted — Workers has no concurrency ceiling.*

### Low

- Cursor pagination encodes `createdAt` at millisecond precision
  (`src/node/lib/pagination.ts:12-14`) vs Postgres microseconds — rows sharing a
  millisecond can be skipped/duplicated at page boundaries. **Carries over — fix regardless**
  (compare on stored value or use UUIDv7).
- `docs/guides/CDK_TEARDOWN.md` references `scripts/destroy-all.sh`, which doesn't exist.
- `as unknown as HandlerResponse` cast (`src/node/handlers/users/update.ts:184`).
- No committed `.env.example` (mitigated by `pnpm init-project` generating env files).
- `DISABLE_BLUE_GREEN` escape hatch exists but is undocumented in AGENTS.md.

## Strengths (what the migration must preserve)

- **Auth:** single shared WorkOS verifier used by Lambda authorizer *and* local dev
  (no prod/local drift); RS256 pinned, issuer + `client_id` binding, JWKS cold-start
  timeout guards.
- **Webhooks:** constant-time signature comparison, timestamp tolerance, payload cap,
  DB-backed idempotency with stale-lock reclaim, DLQ with alarm.
- **Infra:** blue-green canary deploys with alarm-triggered auto-rollback on every
  Lambda; least-privilege IAM (no wildcards); secret ARNs (not values) in env;
  `RETAIN` policies on prod data; cost budgets with forecast alerts.
- **DX:** `init-project.ts` scaffolding, `destructive-db-guard.ts` (type-the-hostname
  confirmation), handler templates, ~3,500 lines of accurate docs incl. threat model
  and SOC 2 checklist.
- **Tests:** real-Postgres transaction-atomicity tests guarding the
  `neon-serverless` vs `neon-http` driver choice; ephemeral DB in CI before every deploy.
